Professional Self Assessment

This program helped me go from just being interested in tech to actually understanding how software works end to end. Not just writing code, but thinking about structure, usability, and how everything connects.

One big thing for me was explaining code clearly. It's just as important as writing it. The code review in this project really pushed that. I had to walk through my app, point out issues, and explain why changes mattered.

I also got more comfortable with problem solving. Adding filtering and sorting to my app seems small, but it made a big difference in how usable it is. It made me think more about how data should actually be handled, not just stored.

Working with full stack apps and databases tied everything together. Frontend, backend, and data all depend on each other. This project helped that click more for me.

Security was another big shift. I started thinking more about bad input, protecting routes, and making sure data is handled safely. Not just building features, but building them the right way.

Overall, this portfolio shows how I improved one project across multiple areas. It reflects how I think now compared to when I started, and gives a solid example of the kind of work I can do going forward.

Code Review

The code review is where this project started. I walked through my original app and explained how everything worked before making any changes.

From there, I pointed out areas to improve. Things like usability, data handling, and security. That gave me a clear plan for what to enhance.

It was useful because it made me think more critically about my own code instead of just adding features.

Original Artifact

The project I used is the Travlr Getaways full stack app from CS 465. It has both a customer site and an admin side for managing trips.

I picked it because it already had a solid base and gave me room to improve it instead of starting over.

It let me show frontend work, backend logic, database use, and security improvements all in one place.

Enhancements

Software Engineering: I cleaned up parts of the admin side and made it easier to use. Nothing crazy, just making it more practical and organized.

Algorithms and Data Structures: I added filtering and sorting so the data is easier to work with. Small change but big improvement in usability.

Databases: I improved CRUD behavior and added validation so bad data does not make it into the database. This made the app more reliable.

Together these changes made the project feel more complete and closer to something you would actually use.
